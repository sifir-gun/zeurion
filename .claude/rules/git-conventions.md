# Conventions git

- Messages de commit au format Conventional Commits : type(scope): description.
- Types utilisés : feat, fix, chore, docs, refactor. Scope quand il apporte de la précision (setup, skills, zeurion, superviseur, context).
- Description à l'impératif ou au substantif, en français, concise. Corps du message pour le contexte quand le titre ne suffit pas.
- Un commit = un changement cohérent. Ne pas mélanger deux sujets dans un même commit.
- Ne jamais réécrire l'historique déjà poussé sur origin.
