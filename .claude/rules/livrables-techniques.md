# Livrables techniques

Le livrable attendu après une décision d'architecture est une spec prête à implémenter, pas un résumé.

- Workflows n8n : décrire les nœuds, les déclencheurs, les données échangées et les cas d'erreur. Cible : instance n8n auto-hébergée.
- Schémas SQL : Postgres + pgvector, schémas séparés par niveau (convention du modèle de données Zeurion, references/usine-business-autonome-v2.md, section 5).
- Toute spec inclut ses critères de validation : comment vérifier que l'implémentation est correcte.
